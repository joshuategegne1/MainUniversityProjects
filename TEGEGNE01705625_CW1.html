1.1 ^ Linear Regression¶

We fit a linear model of Y against X, where Y is a vector that holds the output "LC50" and X is a matrix containing the other 10 covariates as columns and another columns of 1s; $$ Y = X \beta + \epsilon $$ we assume that the residuals are normally distributed and independant. $$ \epsilon_i \stackrel{i.i.d}{\sim} N(0, \sigma^2) \quad i = 1,.., N $$

We find our maximum likelihood estimator $\hat{\boldsymbol \beta}_{ML}$ for $\boldsymbol\beta$ by maximising the log likelihood: $$ L(Y | X, \beta, \sigma^2)= -\frac{N}{2}log(2\pi \sigma^2) - \frac{1}{2 \sigma^2}(Y - X \beta)^T(Y- X \beta) $$ By taking the gradient of the log likelihood and setting it equal to 0, $$ \nabla_{\beta}L(Y | X, \beta) = -\frac{1}{2\sigma^2} (-2 X^T Y + 2(X^T X)\beta)$$ we obtain our maximum likelihood estimator. $$ \hat{\boldsymbol \beta}_{ML} = (X^T X)^{-1} X^T Y$$.

Note that the maximum likelihood estimator also minimises the loss function

$$ L(\beta) = \lVert Y- X\beta \rVert ^2 .$$

We now compute the coefficient of determination, $$ R^2 = 1 - \frac{\lVert Y- X \hat{\boldsymbol \beta} \rVert ^2}{\lVert Y- \bar{Y} \rVert ^2}$$ which is at most 1. The higher $R^2$ is, the more accurate our fitted model is, because it quantifies how much smaller the residual sum of squares is for our model compared to when one parameter is used,and the fitted value of y is set to the mean value of y from the sample data.

def Rsquared(Y, x, beta):
    N, D = x.shape
    X = np.hstack((np.ones((N,1)), x))
    e = Y - X @ beta # residuals
    RSS = e.T @ e # residual sum of squares
    RSS_0 = np.sum((Y- np.mean(Y))**2) # residual sum of squares if beta contains only intercept
    R2 = 1- RSS/RSS_0
    return R2

1.2 ^ Ridge Regression¶

We will now perform ridge regression on the data, where we now try to minimise a new loss function, $$ L_{ridge}(\beta) = \lVert Y- X\beta \rVert ^2 + \lambda \lVert \beta \rVert^2 .$$ where $\lambda > 0$ penalises the estimator if its norm is large. We first define the MSE (mean squared error) of $\hat{\beta}$, an estimator for $\beta$, which measures how accurate an estimator is $$ \text{MSE}(\hat{\beta}, \beta) = \text{E}^2(\hat{\beta} - \beta) + \text{Var}(\hat{\beta}).$$ This should not be confused with the (sample) MSE of the fitted values, $\lVert \hat{Y} - Y \rVert ^2/n$, which we will frequently refer to. Compared to the MLE (minimum likelihood estimator) the ridge estimator has a smaller variance but has a larger bias, the MLE is unbiased. The larger $\lambda$ is, the smaller the variance of the ridge estimator is and the larger its bias. If we can work with this trade-off, we can pick a $\lambda$ where our ridge estimator has a smaller mean squared error. The minimiser of the ridge loss function can be computed explicity: $$ \hat{\boldsymbol \beta}_{ridge} = (X^T X + \lambda I)^{-1} X^T Y $$

def beta_ridge(Y,x,lamda):
    """ Fit linear model of Y against x and compute ridge regression estimator with penalty term lamda"""
    N, D = x.shape
    X = np.hstack((np.ones((N,1)), x))
    return np.linalg.solve(X.T @ X + lamda*np.identity(D+1), (X.T @ Y))

We now perform hyperparameter tuning to find the optimal penalty term $\lambda$ with T-fold Cross Validation. For now we ignore the test data (out of sample data) and use only the training data, and set T= 5. The algorithm is as follows:

  1. Split training data S into T subsets : $S_1, .. S_5$
  2. For every $i = 1,..,5$ , train the model on $\bar{S_i} = S \setminus S_i$ with ridge regression
  3. Use fitted model to predict $S_i$ and compute the MSE against actual values of $S_i$
  4. Compute the mean MSE over all folds
  5. Repeat over range of $\lambda$ values, and observe where MSE is minimised

def cross_validation_score(x_train, Y_train, folds, lamda): #average MSE across T=5 folds
  """ Compute cross validation score across training data, performing ridge regression
  with penalty term lamda """
  scores = []
  for i in range(len(folds)):
    
    # Set fold S_k out of one of the 5 folds, and store its complement
    val_indexes = folds[i]
    train_indexes = list(set(range(Y_train.shape[0])) - set(val_indexes))
    
    x_train_i = x_train[train_indexes, :]
    Y_train_i = Y_train[train_indexes]

    x_val_i = x_train[val_indexes, :] 
    Y_val_i = Y_train[val_indexes]
    
    
    #Compute MSE trained model against actual data
    score_i = score(x_train_i, Y_train_i, x_val_i, Y_val_i, lamda) 
    scores.append(score_i)

  # Return the mean squared error over all 5 folds
  return sum(scores)/len(scores)

1.3 ^ Lasso Regression¶

We now will use Lasso regression, where we minimise the loss function $$ L_{lasso}(\beta) = \lVert Y- X\beta \rVert ^2 + \lambda \sum_{i=1}^p L_c(\beta_i) $$ $$ \text{where} \quad L_c(\beta_i) = \begin{cases} \frac{1}{2}\beta_i^2 & \text{if} |\beta_i| \leq c \\ c(|\beta_i| - \frac{1}{2}c) & \text{otherwise} \\ \end{cases} \quad p = \text{number of parameters}, c = 0.001$$

The penalty term is approximated with Huber functions $L_c$, which are smooth. We cannot find an analytical solution for the minimum of the Lasso loss function, so instead we will use gradient descent. $$ [\nabla_{\beta}L_{\text{lasso}}(\beta)]_i = [2(X^T X)\beta -2 X^T Y]_i + \lambda [\nabla_{\beta}L_c]_i$$ $$ \text{where} [\nabla_{\beta}L_c]_i = \begin{cases} -c & \text{if} \beta_i < -c \\ \beta_i & \text{if} c \leq \beta_i \leq c \\ c & \text{if} \beta_i > c \\ \end{cases}$$

In gradient descent, we first initalise $\beta_0$, and iteratively define $\beta_{k+1} = \beta_{k} - \eta_k \nabla_{\beta} L_{\text{lasso}}(\beta_k)$. The step size $\eta_k$ can be chosen appropiately to speed up convergence- in this case will we set it to be constant for convenience. Ideally, every time $\beta_k$ is updated, its distance from the global minimiser of the Lasso-Huber loss functions reduces- this may not be true if the step size isnt chosen appropiately or the loss function is non-convex.

We will use stochastic gradient descent, a type of gradient descent which instead samples from the data set and computes an approximation to the gradient- this is particularly helpful when working with large datasets. This can result in faster convergence, although it may not accurately reach the actual minimum like standard gradient descent does.

def huber(beta, c=1e-3):
    """ Input beta and c, output value of Huber function"""
    return sum([0.5*i**2 if abs(i) <= c else c*(abs(i)-0.5*c) for i in beta]) 
  
def grad_huber(beta, c=1e-3):
    """ Input beta and c, output gradient of Huber function"""
    return np.array([i if abs(i) <= c else c*np.sign(i) for i in beta])

Now that we have defined the Huber function and its gradient, we can perform stochastic gradient descent, where we will print the cost and mean squared error every time the number of iterations doubles.

c = 1e-3
def minimize_ls_huber(X, y, lambd, max_iterations=10000, stop_criterion=1e-5, batch_size = 16,
                       learning_rate=1e-4, print_outcome=False):
    # initialise zero weights
    beta = np.zeros(X.shape[1]+1).T
    nth = 0
    # initialise starting cost as infinity
    prev_cost = np.inf
    
    # stochastic gradient descent
    indices = np.arange(len(y))
    for iteration in np.arange(1, max_iterations):
        # shuffle to prevent repeating update cycles
        np.random.shuffle(indices)
        batch_idx = indices[:batch_size]
        X_b, y_b = X[batch_idx], y[batch_idx]
        for xi, yi in zip(X_b, y_b):
            
            n = len(xi)
            xi = np.concatenate((np.array([1]), xi))
            XX = np.outer(xi, xi)
            Xy = xi.T * yi  

            ascent = 2*(XX @ beta - Xy) + lambd* grad_huber(beta, c)
            beta = beta - (learning_rate)*ascent

        # convergence check on 2^n'th iteration
        if iteration==2**nth or iteration==max_iterations-1:
            # compute cost
            n, p = X.shape
            X_ = np.hstack((np.ones((n,1)), X))
            mse = np.mean((y- X_ @ beta)**2)  
            cost = mse*n + lambd * huber(beta)
            if print_outcome:
                print("Iteration is: {}, Cost is: {}, MSE is: {}".format(iteration, cost, mse))
            # stop criterion
            if abs(prev_cost - cost) < stop_criterion * prev_cost:
                return beta
              
            prev_cost = cost
            nth += 1
    
    return beta

With some experimentation, we can observe that for a learning rate of 0.0001 and 20,000 iterations, the cost rapidly reduces, and the mean squared error becomes close to the minimum we calculated when we used ridge and linear regression, 0.18. Note that if the learning rate is too large, the cost will oscillate below and above the global minimum.

2.1 ^ kNN Classifier¶

In this section we will use k-nearest neighbours to classify our data. We will work with continuous inputs and discrete ouputs, with data of the form $\{(x^{i}, y^{i}) \} , \: i = 1, \dots ,n $, where $x^i \in \mathbb{R}^{30}$ and $\: y^{i} \in \{0,1\}$. To make a prediction $\hat{y}$ on some input $x_{in}$,we use the following algorithm:

  1. For i = 1,..,n, compute $\lVert x_{in} - x^{i} \rVert $, where $\lVert \rVert$ is the Euclidean metric
  2. Compute k-nearest neighbours of $x_{in}$: $x^{i_1}, \dots , x^{i_n} $, by finding k smallest distances
  3. Compute number of data of these k-nearest neighbours with output 0, or 1
  4. Find class that most neighbours of $x_{in}$ belong to, and assign $\hat{y}$ to the corresponding class.

def standardise(X_train, X_test):
    """Takes in dataset X and outputs normalised data"""
    # Normalise by column/covariate
    mu = np.mean(X_train, 0)
    sigma = np.std(X_train, 0)
    X_train_std = (X_train - mu) / sigma
    
    #Apply same transformation to test data 
    X_test_std = (X_test - mu)/sigma

def k_neighbours(X_train, X_test, k=5, return_distance=False):
  """ Outputs matrix where row i represents the k nearest neighbours of i'th
  data point in X_train to X_test"""
  n_neighbours = k
  dist = []
  neigh_ind = []
  
  # compute distance from each point x_test in X_test to all points in X_train 
  point_dist = [euclidian_distance(x_test, X_train) for x_test in X_test] 

  # determine which k training points are closest to each test point
  for row in point_dist:
      enum_neigh = enumerate(row)
      sorted_neigh = sorted(enum_neigh, key=lambda x: x[1])[:k]

      ind_list = [tup[0] for tup in sorted_neigh]
      dist_list = [tup[1] for tup in sorted_neigh]

      dist.append(dist_list)
      neigh_ind.append(ind_list)
  
  # return distances together with indices of k nearest neighbours
  if return_distance:
      return np.array(dist), np.array(neigh_ind)
  
  return np.array(neigh_ind)

2.2 ^ Random forests¶

Random forests are an extension of decision trees. Our goal is to partition the input domain into subsets $R_1,..,R_k$ for some k, where these subsets mostly contain elements of one class. We can then use the majority rule to make predictions- we observe which region $R_{i}$ some out of sample input $x_{in}$ belongs to, and predict $\hat{y}$ to be the most common class in $R_i.$

To create one such tree, we iteratively perform (j,s) splits: where we randomly pick a predictor j from some strict subset of the predictors, and pick a value s to split the region R into the regions $R_{left} = \{R \mid x^{j} \leq s\}$ and $R_{right} = \{R \mid x^{j} \geq s\}$. We pick s such that the weighted sum of the cross entropy of the two subsets is maximised, which is a function that quantifies how homogenous the subset of the input data is. The cross entropy is defined as follows:

$$ CE(\pi(R)) = -\sum_{q} \pi_q(R) \log(\pi_q(R))$$

where we denote $\pi_q(R)$ to be the proportion of data points in R that belong to class q.

We can create an ensemble of decision trees by creating B models, where each tree is modelled from a random sample of size N' < N. To make a prediction with multiple trees, we instead compute the average vector $\pi(R_i)$ across every tree where $x_{in} \in R_i$ and predict $\hat{y}$ to be the class with the highest corresponding proportion.

Note that if the depth of these trees is too large then overfitting will occur, so we will explore how the hyperparameters tree depth and number of trees affect the model. We do not standardise the data- partitioning data is not affected by standardisation.

def CE(y):
  """ Calculate the cross entropy for subset y"""
  ratio = np.sum(y)/len(y)
  if ratio in [0,1]:
    return 0
  else:
    props = np.array([ratio,1 - ratio])
    return -np.dot(props, np.log(props))

def split_samples(X, y, column, value):
  """ Splits at a given column index and value, outputting the two new subsets""" 

  left_mask = (X[:, column] < value)
  
  X_left, X_right = X[left_mask, :], X[~left_mask, :]
  y_left, y_right = y[left_mask], y[~left_mask]

  return (X_left, X_right), (y_left, y_right)

def CE_split_value(X, y, column):
  """Calculate the cross entropy based on a given column with the split that minimizes the CE."""
  
  unique_vals = np.unique(X[:, column])

  assert len(unique_vals) > 1, f"There must be more than one distinct feature value. Given: {unique_vals}."

  CE_val, threshold = np.inf, None
  
  # split the values of i-th feature and calculate the cost 
  for value in unique_vals:
    (X_l, X_r), (y_l, y_r) = split_samples(X, y, column, value) 

    # if one of the two sides is empty, skip this split.
    if len(y_l) == 0 or len(y_r) == 0:
      continue
    
    p_left = len(y_l)/(sum(y_l) + sum(y_r))
    p_right = 1 - p_left
    new_cost = p_left * CE(y_l) + p_right * CE(y_r) 
    if new_cost < CE_val:
      CE_val, threshold = new_cost, value
    
  return CE_val, threshold

  """
  Choose the best feature to split according to criterion.
  Args:
      n_features: number of sampled features.
      X: training features, of shape (N, D).
      y: vector of training labels, of shape (N,).
      sample_weights: weights for each samples, of shape (N,).
      columns_dict: a dictionary mapping column indices to whether the column is categorical or numerical variable.
  Returns:
      (float, int, float): the minimized gini-index, the best feature index and value used in splitting.
  """
  columns_dict = {index: False for index in range(X_train.shape[1])}
  columns = np.random.choice(list(columns_dict.keys()), n_features, replace=False)
  columns_dict = {c: columns_dict[c] for c in columns}

  min_CE_index, split_column, split_val = np.inf, 0, 0
  for column, categorical in columns_dict.items():
    # skip column if samples are not seperable by that column.
    if len(np.unique(X[:, column])) < 2:

    # search for the best splitting value for the given column.
    CE_index, val = CE_split_value(X, y, column)

def predict_rf(rf, X):
  """
  Predict classification results for X.
  Args:
      rf: A trained random forest through train_rf function.
      X: (pd.Dataframe) testing sample features, of shape (N, D).
  Returns:
      (np.array): predicted testing sample labels, of shape (N,).
  """
  # Aggregate over all models

  def aggregate(decisions):
    count = defaultdict(int)
    for decision in decisions:
      count[decision] += 1
    return max(count, key=count.get)

  
  if len(X.shape) == 1:
      return aggregate([classify(tree, X) for tree in rf])
  else:
      return np.array([aggregate([classify(tree, x) for tree in rf]) for x in X])

def rf_score(rf, X_test, y_test):
  y_pred = predict_rf(rf, X_test)
  return np.mean(y_pred==y_test)

The true positve rate(or recall), precision and F-score are defined as follows:

$$ \text{recall} = \frac{TP}{TP+FN} \quad \text{precision} = \frac{TP}{TP+FP} \quad F = 2 \frac{\text{precision}*\text{recall}}{\text{precision} + \text{recall}}$$

These quantities range from 0 to 1, and the larger they are, the more accurate the model is. We compute the recall and F-scores for our training and test data.

def prediction_rates(y_pr, y_ac):
    # true positives
    tp = np.sum(np.logical_and(y_pr == 1, y_ac == 1))
    # true negatives
    tn = np.sum(np.logical_and(y_pr == 0, y_ac == 0))
    # false positives
    fp = np.sum(np.logical_and(y_pr == 1, y_ac == 0))
    # false negatives
    fn = np.sum(np.logical_and(y_pr == 0, y_ac == 1))
    #recall
    recall = tp / (tp + fn) 
    #predict
    precision = tp / (tp + fp) 
    F_score = 2*(precision*recall)/(precision+recall)
    return recall, F_score

2.3 ^ Support Vector Machine¶

In this section we use soft-margin SVMs (support vector machines), where our goal is to find a hyperplane $\boldsymbol w \cdot x_i + b$ that can split the input domain into two subsets that are as homogenous in class as possible - the output is 1 or -1. We do this my minimising the loss function $$ \mathcal L (\boldsymbol w) = \frac{1}{2} \| \boldsymbol w \|^2 + \frac{\lambda}{n} \sum_{i=1}^n \max \bigg( 0, 1-y_i (\boldsymbol w \cdot x_i + b) \bigg) \, . $$

When performing soft-margin SVM's the hardness parameter $\lambda$ controls the penalty term that measures how imperfect the seperation of the data is. If we could split our data into two seperate classes with one plane, then $y^{(i)}(x^{(i)}*w + b) \geq 1$ for all points, meaning that the loss function would just $L(w) = \frac{1}{2} \lVert w \rVert^2$. The larger the hardness parameter is, the more the constraint violations are penalised and the more representative it becomes of hard-margin SVM.

def compute_cost(w, X, y, hardness=1e5):
    distances = 1 - y * (X @ w) 
    distances[distances < 0] = 0  #  max(0, distance)
    hinge = hardness * distances.mean()
    # calculate cost
    return 0.5 * np.dot(w, w) + hinge

The score function checks the accuracy of our predictions. We look at what side of the hyperplane $x_{in}$ is by computing the sign of $x_{in} \cdot w$. If it is positive, $\hat{y} = 1$, otherwise $\hat{y}= -1.$

def score(w, X, y):
    y_preds = np.sign(X @ w)
    return np.mean(y_preds == y)

The minimum to the loss function cannot be analytically found so we use stochastic gradient descent. We set our step size $\eta_k$ to be equal to $\frac{C}{k}$, where C is some constant (the learning rate) and k is the number of iterations. This prevents the loss function from rapidly oscillating above and below the global optimum, which may happen otherwise if a fixed step size is used. Whenever the iteration number is of the form $2^k$, we will also output the cost and accuracy.

def calculate_cost_gradient(w, X_batch, y_batch, hardness=1e5):
    # if only one example is passed
    if type(y_batch) == np.int32:
        y_batch = np.asarray([y_batch])
        X_batch = np.asarray([X_batch])  # gives multidimensional array

    distance = 1 - (y_batch * (X_batch @ w))
    dw = np.zeros(len(w))

    for ind, d in enumerate(distance):
        if max(0, d)==0:
            di = w
        else:
            di = w - (hardness * y_batch[ind] * X_batch[ind])
        dw += di

    return dw/len(y_batch)  # average

def sgd(X, y, batch_size=16, max_iterations=5000, stop_criterion=0, learning_rate=7.5e-4, hardness=1e5, print_outcome=False):
    # initialise zero weights
    weights = np.zeros(X.shape[1])
    nth = 0
    # initialise starting cost as infinity
    prev_cost = np.inf
    
    # stochastic gradient descent
    indices = np.arange(len(y))
    for iteration in np.arange(1, max_iterations):
        # shuffle to prevent repeating update cycles
        np.random.shuffle(indices)
        batch_idx = indices[:batch_size]
        X_b, y_b = X[batch_idx], y[batch_idx]
        for xi, yi in zip(X_b, y_b):
            ascent = calculate_cost_gradient(weights, xi, yi, hardness) 
            weights = weights - (learning_rate/iteration)*ascent

        # convergence check on 2^n'th iteration
        if iteration==2**nth or iteration==max_iterations-1:
            # compute cost
            cost = compute_cost(weights, X, y, hardness)
            if print_outcome:
                print("Iteration is: {}, Cost is: {}, Score is {}".format(iteration, cost, score(weights, X, y) ))
            # stop criterion
            if abs(prev_cost - cost) < stop_criterion * prev_cost:
                return weights
              
            prev_cost = cost
            nth += 1
    
    return weights

def cross_validation_score(x_train, Y_train, folds, hardness, max_iterations=5000): #average accuracy across T=5 folds
  """ Compute cross validation score across training data, performing SVM classification
  with penalty term lamda """
  scores = []
  for i in np.arange(len(folds)):
    
    # Set fold S_k out of one of the 5 folds, and store its complement
    val_indexes = folds[i]
    train_indexes = list(set(range(Y_train.shape[0])) - set(val_indexes))
    
    x_train_i = x_train[train_indexes, :]
    Y_train_i = Y_train[train_indexes]

    x_val_i = x_train[val_indexes, :] 
    Y_val_i = Y_train[val_indexes]
    
    weights = sgd(x_train_i, Y_train_i, batch_size=16, hardness = hardness, 
                  max_iterations = max_iterations,print_outcome=False)    
    score_i = score(weights, x_val_i, Y_val_i)
    
    scores.append(score_i)

  # Return the mean squared error over all 5 folds
  return sum(scores)/len(scores)

def roc_curve(x_train, y_train, x_test, y_test, hardness):
    
  tpr_l, fpr_l = [], []
    
  #Train model on training data with given hardness
  weights = sgd(x_train, y_train, batch_size=32, max_iterations=1000, stop_criterion=0.01, 
        learning_rate=7.5e-4, hardness=hardness, print_outcome=False)

  #Predict on test data
  Y_prob = x_test @ weights
    
  #Convert output classes from 1s and 0s to 1s and -1s to make calculations easier
  Y_test = np.array([1 if i==1 else 0 for i in y_test])
    
  thresholds = reversed(sorted(set(Y_prob)))   
  for threshold in thresholds:
    # Apply thresholding
    y_thresholded = (Y_prob >= threshold)

    # true positives
    tp = np.sum(Y_test & y_thresholded)
    # true negatives
    tn = np.sum((~Y_test) & (~y_thresholded))
    # false positives
    fp = np.sum((~Y_test) & y_thresholded)
    # false negatives
    fn = np.sum(Y_test & (~y_thresholded))

    tpr = tp / (tp + fn) 
    fpr = fp / (tn + fp) 
    tpr_l.append(tpr)
    fpr_l.append(fpr)

  # Compute AUC using Trapezoidal rule
  auc = np.trapz(tpr_l, fpr_l)
  return auc, tpr_l, fpr_l, thresholds
